# Skill: git-operation 设计文档

## 一、基本信息

| 项目 | 内容 |
|------|------|
| 职责 | 执行Git操作 |
| 输入 | 变更内容 + 操作类型 |
| 输出 | 操作完成确认 |

---

## 二、设计讨论记录

### 2026-01-29 讨论

**确定的要点：**
- 分支策略：简化版 GitHub Flow
- 提交规范：`<类型>(<范围>): <描述>`
- 检查点：每个任务验证通过后提交

---

## 三、方法论层

### 提交时机

| 时机 | 操作 |
|------|------|
| 任务完成且验证通过 | commit |
| 功能完成且集成测试通过 | push |
| 开始新功能 | 创建feature分支 |
| 功能完成 | merge到main |

### 提交信息生成

基于变更内容自动生成：
- 新增文件 → feat
- 修改文件 → fix/refactor
- 文档变更 → docs
- 测试变更 → test

---

## 四、操作指南层

### Commit流程

```
1. 检查变更
2. 确认验证已通过
3. 生成提交信息
4. git add .
5. git commit
```

### Push流程

```
1. 确认集成测试通过
2. 确认分支正确
3. git push
```

---

## 五、边界处理层

| 情况 | 处理 |
|------|------|
| 冲突 | 通知人处理 |
| 验证未通过 | 拒绝提交 |
| push失败 | 检查原因，通知 |
