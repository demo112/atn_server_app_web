# Skill: project-logging 设计文档

## 一、基本信息

| 项目 | 内容 |
|------|------|
| 职责 | 记录项目级别的日志 |
| 输入 | 事件类型 + 事件内容 |
| 输出 | 日志记录 |

---

## 二、设计讨论记录

### 2026-01-29 讨论

**日志类型：**
- 进度日志：任务状态
- 决策日志：技术/业务决策
- 问题日志：遇到的问题
- 变更日志：代码变更

---

## 三、方法论层

### 日志设计原则

| 原则 | 说明 |
|------|------|
| 及时 | 事件发生时记录 |
| 完整 | 包含必要上下文 |
| 结构化 | 便于查询 |

---

## 四、操作指南层

### 流程

```
1. 识别事件类型
2. 确定记录位置
3. 格式化内容
4. 追加到日志文件
```

---

## 五、边界处理层

| 情况 | 处理 |
|------|------|
| 文件不存在 | 创建 |
| 格式不一致 | 规范化 |
