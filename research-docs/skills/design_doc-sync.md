# Skill: doc-sync 设计文档

## 一、基本信息

| 项目 | 内容 |
|------|------|
| 职责 | 保持文档与代码同步 |
| 输入 | 代码变更 + 触发事件 |
| 输出 | 更新后的文档 |

---

## 二、设计讨论记录

### 2026-01-29 讨论

**核心原则：** 文档是人机信息对接的核心，必须保持同步。

**触发规则：**
| 触发事件 | 更新文档 |
|---------|---------|
| 需求分析完成 | requirements/*.md |
| 设计完成 | design/*.md |
| API变更 | api/*.md |
| 数据库变更 | database/*.md |
| 任务状态变化 | progress/*.md |
| 做出决策 | decisions/*.md |
| 遇到/解决问题 | issues/*.md |
| 代码提交 | changelog.md |

---

## 三、方法论层

### 文档同步原则

| 原则 | 说明 |
|------|------|
| 事件驱动 | 变更触发更新 |
| 增量更新 | 只改变化的部分 |
| 格式统一 | 遵循模板 |

---

## 四、操作指南层

### 流程

```
1. 识别变更类型
2. 确定需更新的文档
3. 读取当前文档
4. 更新相关内容
5. 保存文档
```

---

## 五、边界处理层

| 情况 | 处理 |
|------|------|
| 文档不存在 | 创建新文档 |
| 无法确定更新内容 | 标记待补充 |
