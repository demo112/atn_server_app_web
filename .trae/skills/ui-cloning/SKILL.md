## 第一阶段：截图分析与设计定位

### 1.1 截图采集

从目标产品截取界面截图：
- 每个核心页面单独截图
- 包含不同状态（空态、加载态、数据态）
- 注意截取交互细节（弹窗、下拉等）

### 1.2 设计分析（关键步骤）

**生成前必须分析目标 UI 的设计语言：**

| 分析维度 | 问题 | 示例答案 |
|----------|------|----------|
| 整体风格 | 极简/复杂/扁平/拟物？ | 扁平化、卡片式布局 |
| 配色方案 | 主色/辅色/强调色？ | 蓝色主调 #1890FF |
| 字体风格 | 衬线/无衬线/粗细对比？ | 无衬线、标题加粗 |
| 间距节奏 | 紧凑/宽松/模块化？ | 16px 基础间距 |
| 特色元素 | 圆角/阴影/渐变/动效？ | 大圆角、轻阴影 |

## 第二阶段：Stitch 生成 UI 设计

### 2.1 基础生成

1. 访问 https://stitch.withgoogle.com
2. 选择模式：Web 或 Mobile
3. 上传截图 + 输入提示词

### 2.2 提示词模板

```
参考图片中的界面，设计一个 [功能描述] 页面。

【设计要求】
- 风格：[从分析中提取，如"扁平化、卡片式"]
- 配色：主色 [#xxx]，强调色 [#xxx]
- 圆角：[大圆角 12px / 小圆角 4px]
- 间距：基础间距 [16px]

【功能需求】
1. [功能点1]
2. [功能点2]
3. [功能点3]

【差异化】
- 保留原设计的 [xxx 特点]
- 优化 [xxx 部分] 的视觉层次
```

### 2.3 迭代优化

- **文字迭代**：用自然语言描述修改需求
- **变体生成**：生成多个设计方案对比
- **原型连接**：将多个页面串联成可交互流程

## 第三阶段：代码生成与品质提升

### 3.1 导出选项

| 格式 | 用途 |
|------|------|
| HTML/CSS | 快速预览 |
| React | 技术团队集成 |
| Figma | 设计师调整 |

### 3.2 品质提升提示词

在 AI Studio 或 Claude 中使用：

```
将这个 UI 代码转换为高质量的 [React/React Native] 组件。

【设计提升要求】
1. 字体：避免 Inter/Arial 等通用字体，使用 [指定字体]
2. 配色：使用 CSS 变量统一管理，主色突出
3. 动效：添加微交互（按钮 hover、列表进入动画）
4. 间距：使用 8px 网格系统，保持一致性
5. 层次：通过阴影/透明度建立视觉层次

【代码质量要求】
1. 组件化：拆分为可复用的小组件
2. 类型安全：完整的 TypeScript 类型定义
3. 响应式：适配不同屏幕尺寸
4. 无障碍：添加 aria 标签

【避免】
- 通用的 AI 生成风格（紫色渐变、白色背景）
- 过于均匀的配色分布
- 缺乏层次的扁平布局
```

## 设计品质检查清单

### 视觉层面
- [ ] 字体选择是否独特（非 Inter/Arial/Roboto）
- [ ] 配色是否有主次之分
- [ ] 间距是否遵循网格系统
- [ ] 是否有适当的阴影/层次感
- [ ] 圆角是否统一

### 交互层面
- [ ] 按钮是否有 hover/active 状态
- [ ] 列表是否有进入动画
- [ ] 加载状态是否有骨架屏
- [ ] 空状态是否有引导

### 代码层面
- [ ] 是否使用 CSS 变量管理主题
- [ ] 组件是否足够小且可复用
- [ ] 是否有完整的 TypeScript 类型
- [ ] 是否考虑响应式

## 最佳实践

1. **先分析后生成** - 花 5 分钟分析目标 UI 的设计语言
2. **参考图优先** - 上传截图比纯文字描述效果更好
3. **分页面生成** - 每个页面单独生成，避免一次性过多
4. **品质提升** - 生成后用提示词二次优化代码品质
5. **保持独特** - 避免生成"AI 味"的通用界面
