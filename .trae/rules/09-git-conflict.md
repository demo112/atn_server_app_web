# Git 冲突处理规范

## 预防优先
- 通过模块分工避免冲突
- 开始改文件前先沟通
- 频繁 push，减少本地积压
- 每次开始工作前先 `git pull`

## 冲突处理
- AI 分析冲突并提出解决方案
- **AI 判断必须完全基于 `.trae/specs/{feature}/requirements.md`**
- 禁止无依据猜测
- 用户确认后执行

| 冲突类型 | 处理方式 |
|----------|----------|
| 简单冲突 | AI自动合并，用户确认 |
| 复杂冲突 | AI列出选项，用户选择 |
| 无法判断 | 暂停，请求用户补充需求 |

**禁止用户自行解决代码冲突**
