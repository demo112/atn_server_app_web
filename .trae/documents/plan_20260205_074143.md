# Web UI 自动化测试实施计划 (用户管理模块)

## 1. 目标
基于 `20260205-user-creation-validation-fix` 修复记录，实现用户创建功能的 UI 自动化测试，验证前端表单校验逻辑。

## 2. 实施范围
- **新增 Page Object**: `packages/e2e/pages/user.page.ts`
- **新增测试文件**: `packages/e2e/tests/user/create-user.spec.ts`

## 3. 详细步骤

### Step 1: 基础设施准备 (Page Object)
在 `packages/e2e/pages/user.page.ts` 中实现：
- **定位器**: 添加按钮、模态框表单项（用户名、密码、角色等）、提交按钮、错误提示信息。
- **操作方法**:
    - `goto()`: 导航至用户列表页。
    - `openAddModal()`: 点击添加按钮。
    - `fillForm(data)`: 填写表单。
    - `submit()`: 点击确定。
    - `getValidationErrors()`: 获取页面上的错误提示文本。

### Step 2: 编写测试用例
在 `packages/e2e/tests/user/create-user.spec.ts` 中实现：
- **Case 1: 校验失败场景 (Regression)**
    - 输入用户名 "11" (长度<3)。
    - 提交表单。
    - **断言**: 界面显示 "用户名至少需要3个字符"。
- **Case 2: 正常创建场景 (Happy Path)**
    - 输入合法用户名和密码。
    - 提交表单。
    - **断言**: 模态框关闭，列表中出现新用户。

### Step 3: 执行与验证
- 运行 `npx playwright test tests/user/create-user.spec.ts`。
- 确保测试在无头模式下通过，证明修复有效且无回归。
