# 修复方案：修正参数校验并重启服务

## 问题原因
1.  **服务停止**：日志显示 `net::ERR_CONNECTION_REFUSED`，确认为开发环境所有服务（Server/Web/App）已停止运行。
2.  **400 错误**：用户反馈的 `AxiosError: Request failed with status code 400` 很可能源于 `User` 模块的参数校验问题。`user.dto.ts` 中 `page` 和 `pageSize` 字段使用了 `z.string().transform(Number)`，这导致如果不传参数（undefined）或某些情况下类型不匹配时会校验失败。

## 实施步骤
1.  **优化代码**：修改 `packages/server/src/modules/user/user.dto.ts`，将分页参数校验改为更健壮的 `z.coerce.number()`，使其能同时兼容字符串和数字输入，并正确处理默认值。
2.  **重启服务**：
    - 启动 Server (`pnpm --filter @attendance/server dev`)
    - 启动 Web (`pnpm --filter @attendance/web dev`)
    - 启动 App (`npm run android`)
3.  **验证**：服务启动后，观察 Server 终端日志。如果仍有 400 错误，日志中将显示详细的 Zod 校验错误信息，便于进一步排查。

## 验证标准
- 所有服务正常启动。
- Web 端访问用户列表不再报错 400。
