# 修复方案：修正前端删除部门的响应校验

## 问题根因
1.  **后端响应**：后端 `DepartmentController.delete` 返回 `{ success: true, data: null }`。
2.  **前端校验**：前端 `departmentService.deleteDepartment` 使用 `z.void()` 校验响应数据。
3.  **冲突**：Zod 的 `z.void()` 严格要求数据为 `undefined`，而实际接收到的是 `null`，导致校验失败并抛出 "Invalid input: expected void, received null"。

## 实施步骤
1.  **修改前端 Service**：编辑 `packages/web/src/services/department.ts`。
2.  **放宽校验规则**：将 `deleteDepartment` 方法中的 `z.void()` 修改为 `z.union([z.void(), z.null()])`，使其能同时接受 `undefined` 和 `null`。

## 验证标准
- Web 端删除部门操作不再弹出校验错误提示。
- 删除操作能正常完成。
