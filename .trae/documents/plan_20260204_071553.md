# 修复方案：修正数据库表名引用错误

## 问题根因
通过分析 Server 端日志，发现核心错误为 `Table 'ai_atnd.Employee' doesn't exist`。
这是因为在 `AttendanceClockService` 中使用 `prisma.$queryRaw` 执行原生 SQL 时，错误地使用了 Prisma 模型名称 `Employee`，而实际数据库表名是 `employees`（由 `schema.prisma` 中的 `@@map("employees")` 定义）。

这导致打卡及相关核心功能直接崩溃，进而导致 App/Web 端收到错误响应格式（ZodError），造成“几乎所有功能报错”的现象。

## 实施步骤
1.  **修复代码**：修改 `packages/server/src/modules/attendance/attendance-clock.service.ts`，将原生 SQL 中的 `Employee` 替换为 `employees`。
2.  **验证修复**：
    - 运行 `npm run test --filter @attendance/server` 验证修复结果。
    - 检查 Server 终端日志，确认不再抛出该数据库错误。

## 验证标准
- Server 端不再抛出 "Table doesn't exist" 错误。
- 单元测试通过。
