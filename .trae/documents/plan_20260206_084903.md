# E2E 架构统一与解耦计划

针对您关于 "App E2E" 的询问，我已更新计划，确保三端（Server/Web/App）的测试架构统一且解耦。

## 1. 架构解耦 (Configuration Refactor)
**目标**：移除 `packages/e2e` 对 `server` 的隐式依赖，确保测试环境独立。

**实施方案**：
- 修改 `packages/e2e/playwright.config.ts`：
  - 移除读取 `../server/.env` 的代码。
  - 使用 `process.env.PORT || 3001` 并通过 `env` 注入到 Server 进程，确保端口一致。

## 2. 规范确立与 App 规划 (Documentation Update)
**目标**：在 `.trae/rules/11-testing.md` 中明确 **"One E2E Package"** 策略，涵盖 App 端。

**实施方案**：
- **唯一路径**：明确 `packages/e2e` 是 Web 和 App E2E 测试的唯一存放地。
- **App 测试策略**：
  - 规定 App 测试代码也必须存放于此（如 `packages/e2e/tests/app`）。
  - 明确解耦原则：App 测试必须针对**构建后的应用**（APK/IPA 或 Expo Go）进行黑盒测试，**禁止**引用 `packages/app` 源码。
  - 推荐技术栈：Playwright (Experimental Android) 或 Maestro/Detox，保持在 `packages/e2e` 内运行。

## 3. 验证 (Verification)
- 运行 `pnpm test -- --list` 验证 Web E2E 配置的有效性。
- （可选）如果需要，我可以为您初始化 App 测试的目录结构（如创建 `tests/app` 占位符）。
