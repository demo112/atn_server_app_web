# AI编码能力图谱（Trae IDE）

## 一、核心模式

| 模式 | 定位 | 自主程度 |
|------|------|---------|
| Chat模式 | 对话问答 | 被动响应 |
| Builder模式 | 多文件开发 | 半自主（需确认） |
| SOLO模式 | 全自主开发 | 高度自主（从需求到部署） |

---

## 二、能力层次

### 第一层：基础能力
- 代码补全（Cue）
- 代码解释
- 代码生成
- 错误诊断

### 第二层：上下文能力
- `#文件/文件夹` 引用
- 多模态输入（图片）
- URL引用
- 代码库索引
- Web搜索

### 第三层：规则系统（Rules）
- 位置：`.trae/rules/`
- 类型：个人规则（user_rules.md）、项目规则（project_rules.md）
- 作用：约束AI行为、统一代码风格
- 加载：启动时全量加载

### 第四层：自定义Agent
- 定义专业角色
- 配置触发方式（@agent名称）
- 指定可用工具和Skill
- 设定行为规范

### 第五层：Skill（技能）
- 位置：`.trae/skills/`
- 结构：SKILL.md + 脚本 + 参考资料
- 作用：标准化执行流程
- 加载：按需动态加载（三层加载机制）

### 第六层：MCP集成
- 连接外部工具、数据库、API
- 内置MCP市场
- 每个Agent可分配不同MCP服务

### 第七层：多Agent协作
- 内置专业Agent
- 自定义Agent可作为子Agent
- Agent之间可协作

---

## 三、三者协作关系

```
Rules（规则）- "宪法"，不可违背的底线
    ↓ 约束
Agent（角色）- "专家"，特定领域的人格
    ↓ 调用
Skill（技能）- "手册"，标准化的执行流程
```

| 层级 | 职责 | 加载时机 |
|------|------|---------|
| Rules | 定义"什么不能做"和"必须怎么做" | 启动时全量加载 |
| Agent | 定义"我是谁"和"我擅长什么" | 切换角色时加载 |
| Skill | 定义"这件事怎么做" | 任务匹配时按需加载 |

---

## 四、能力边界

### 强项
- UI/前端生成质量高
- 视觉效果好
- 环境配置自动化
- 错误自修复

### 弱项
- 业务逻辑实现不完整
- 会声称"完成"但实际未完成
- 后端复杂逻辑需要更多引导
- 复杂问题可能降级处理

---

## 五、使用原则

1. Rules 要稳定 - 一旦确定，尽量不改
2. Agent 要专注 - 每个Agent专注一个领域
3. Skill 要原子 - 每个Skill只做一件事
4. 必须验证 - 不要相信AI说"完成了"
