# 需求规格说明书：用户登录信息展示与登出

## 1. 背景
当前系统用户登录后，界面顶部未清晰展示当前登录用户的身份信息，且缺乏明显的登出入口，导致用户无法确认身份或安全退出系统。

## 2. 需求概述
在系统顶部导航栏（Header）区域增加用户信息展示区域和登出按钮。

## 3. User Stories

### Story 1: 查看登录信息
**As a** 登录用户, **I want** 在页面顶部看到我的用户名, **So that** 我确认当前登录的身份

**Acceptance Criteria:**
- [ ] AC1: 显示用户名
  - **Given**: 用户已登录
  - **When**: 进入系统主页
  - **Then**: 顶部导航栏右侧显示当前登录用户的姓名（优先显示 name，无 name 显示 username）
- [ ] AC2: 显示角色
  - **Then**: 显示用户的角色标签（如"管理员"或"员工"）

### Story 2: 退出登录
**As a** 登录用户, **I want** 点击退出按钮, **So that** 我可以安全注销并返回登录页

**Acceptance Criteria:**
- [ ] AC1: 登出操作
  - **Given**: 用户已登录
  - **When**: 点击顶部的"退出"按钮
  - **Then**: 
    1. 清除本地登录凭证（Token）
    2. 跳转至登录页面
    3. 提示"已退出登录"（可选）

## 4. 约束条件
- 必须基于现有的 `AuthContext` 和 `LoginVo` 数据结构
- 样式需适配现有的蓝色顶部导航栏风格
