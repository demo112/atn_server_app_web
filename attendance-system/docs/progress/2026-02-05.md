# 进展日志：App 端请假页面增加员工选择

## 📅 2026-02-05

### ✅ 完成事项

#### 1. App 端请假页面支持管理员选择员工 (UA1)
- **需求**: App添加请假的页面没有选择用户，需要添加选择员工（管理员专用）。
- **实现**:
  - 修改 `LeaveScreen.tsx`，引入 `getEmployees` 和 `useAuth`。
  - 实现管理员专属的员工选择 UI（Modal + FlatList）。
  - 更新提交逻辑，管理员可提交选中员工的请假申请，普通员工默认提交自己的。
  - 更新 `LeaveScreen.test.tsx`，增加管理员选择员工的测试用例。
- **验证**:
  - 单元测试 `npm test -- LeaveScreen` 通过。
  - 验证了管理员和普通用户的不同流程。

### 📝 决策记录
- **权限控制**: 仅管理员可见员工选择入口，普通用户无感知。
- **交互设计**: 使用 Modal 弹窗选择员工，避免页面跳转复杂度。
- **默认行为**: 若管理员未选择员工，逻辑上会 fallback 到 0 或校验失败（当前 UI 强制选择或默认不选提交会失败，符合预期）。
