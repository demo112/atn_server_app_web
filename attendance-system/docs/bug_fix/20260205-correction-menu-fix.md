# 补签菜单文案及链接修复记录

## 问题描述
- **现象**：
  1. 左侧菜单“异常处理”文案不符合用户习惯，应为“补签记录”。
  2. 缺少“补签处理”菜单入口，无法直接访问补签处理页面。
- **复现步骤**：
  1. 登录 Web 端。
  2. 展开“考勤管理”菜单。
  3. 观察到“异常处理”菜单项。
  4. 未找到“补签处理”菜单项。
- **影响范围**：
  - Web 端左侧导航菜单。

## 设计锚定
- **所属规格**：SW62 (考勤补签)
- **原设计意图**：提供补签记录查询和补签处理功能。
- **当前偏离**：
  - 菜单文案“异常处理”较为宽泛，不准确。
  - 遗漏了补签处理页面的入口配置。

## 根因分析
- **直接原因**：`Sidebar.tsx` 中菜单配置项文案为“异常处理”，且缺少 `correction-processing` 的配置项。
- **根本原因**：前端菜单配置未及时跟进需求变更或遗漏配置。
- **相关代码**：`packages/web/src/components/layout/Sidebar.tsx`

## 修复方案
- **修复思路**：
  1. 修改 `Sidebar.tsx` 中的菜单配置。
  2. 将 label 从“异常处理”改为“补签记录”。
  3. 新增“补签处理”菜单项，链接至 `/attendance/correction-processing`。
- **改动文件**：
  - `packages/web/src/components/layout/Sidebar.tsx`
