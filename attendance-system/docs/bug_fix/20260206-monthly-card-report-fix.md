# 月度考勤卡表修复记录

## 问题描述
- **现象**：
  1. 月度考勤卡表页面年份无法选择（只能选月份）。
  2. 可能存在 500 错误（与月度汇总报表同源）。
- **复现步骤**：
  1. 进入统计报表 -> 月度考勤卡表。
  2. 尝试跨年查询（失败）。
- **影响范围**：月度考勤卡表功能不可用。

## 设计锚定
- **所属规格**：SW62 (考勤统计)
- **原设计意图**：提供月度考勤明细卡片视图，支持按月筛选。
- **当前偏离**：
  1. 前端 `input type="month"` 交互不符合预期。

## 根因分析
- **年份选择**：
  - **直接原因**：`<input type="month">` 在部分环境下交互受限，不支持直观的年份切换。
- **500 错误**：
  - **原因**：该页面调用的 `getDepartmentSummary` 接口已在之前的修复中解决了 BigInt 序列化问题。本次仅需关注前端修复。

## 修复方案
- **前端**：
  - `MonthlyCardReport.tsx`: 将 `<input type="month">` 替换为独立的年份和月份选择器。

- [x] 原问题已解决：前端可正常选择年份和月份。
- [x] 编译通过：`npm run build` (Web & Server) 通过。
