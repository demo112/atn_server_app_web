# 启动错误修复记录

## 问题描述
- **现象**：浏览器访问 `http://localhost:5173/` 报错 `net::ERR_ABORTED`。
- **复现步骤**：
  1. 在 `atn_server_app_web` 根目录运行 `npm run dev` (Terminal 68)。
  2. 在 `attendance-system` 目录运行 `npm run dev` (Terminal 64)。
  3. 导致端口 3000 冲突或 Vite 进程不稳定。
- **影响范围**：开发环境无法启动。

## 设计锚定
- **所属规格**：开发环境配置
- **原设计意图**：应在 `attendance-system` (Monorepo Root) 下运行 `npm run dev` 或 `npm run dev:services` 来启动所有服务。
- **当前偏离**：存在两个并行的开发服务器进程，其中一个在错误的根目录（或非预期的上下文）运行。

## 根因分析
- **直接原因**：端口/资源冲突导致服务启动失败或不稳定。
- **根本原因**：操作失误，开启了多个重叠的开发环境终端。
- **相关代码**：`package.json` scripts。

## 修复方案
- **修复思路**：停止所有冲突进程，清理环境，重新启动单一的开发服务。
- **改动文件**：无代码改动，仅操作环境。

## 验证结果
- [x] 原问题已解决：Web 服务成功启动在 `http://localhost:5173/`。
- [x] 回归测试通过：Server 服务成功启动在 `port 3000`。
- [x] 设计一致性确认：符合单入口启动原则。
