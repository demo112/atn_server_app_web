# 修复人员管理默认选中部门问题

## 问题描述
- **现象**：用户反馈在添加人员时，需要手动选择部门，没有默认选中左侧部门架构中已选的部门。
- **复现步骤**：
  1. 进入人员管理页面。
  2. （可选）在左侧部门树选择一个部门。
  3. 点击“添加人员”。
  4. 弹窗中的“部门”字段可能为空，或者如果是已选部门则显示为只读，用户体验不佳。
- **影响范围**：人员管理页面添加人员功能。

## 设计锚定
- **所属规格**：UA1 (User Administration) / SW62 (人员管理)
- **原设计意图**：虽然 Design 文档未明确指定交互细节，但在上下文中，添加操作通常应继承当前上下文（即当前选中的部门）。
- **当前偏离**：
  1. 当未选中部门时，弹窗未提供默认值。
  2. 当选中部门时，弹窗将部门字段设为只读，限制了用户修改（虽然是默认值，但用户可能希望修改）。

## 根因分析
- **直接原因**：`EmployeeModal` 组件在初始化 `formData` 时，仅在“只有一个部门”的特殊情况下才设置默认值。且存在只读逻辑限制了交互。
- **根本原因**：前端组件逻辑未考虑到“未选中部门时默认使用根部门”以及“允许修改默认部门”的交互需求。
- **相关代码**：`packages/web/src/pages/employee/components/EmployeeModal.tsx`

## 修复方案
- **修复思路**：
  1. 修改初始化逻辑：如果没有传入 `defaultDeptId`，则默认选中部门树的第一个节点（通常是根部门）。
  2. 移除只读逻辑：即使有默认值，也允许用户通过下拉框修改部门。
- **改动文件**：
  - `packages/web/src/pages/employee/components/EmployeeModal.tsx`

## 验证结果
- [x] 原问题已解决：逻辑分析确认，未选中部门时将默认选中第一个部门；选中部门时将预填该部门且可修改。
- [x] 回归测试通过：Type check 通过。
- [x] 设计一致性确认：符合通用交互设计原则。
