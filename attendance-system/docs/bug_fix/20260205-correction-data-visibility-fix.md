# 补签处理数据不可见问题修复记录

## 问题描述
- **现象**：用户反馈"现在补签处理数据查不到了"。
- **复现步骤**：
  1. 进入补签处理页面
  2. 默认筛选条件下列表为空
- **影响范围**：补签处理页面 (CorrectionProcessingPage)

## 根因分析
- **直接原因**：
  1. 上一次修复中将默认状态过滤设置为 "异常" (abnormal)，隐藏了 "正常" (normal) 记录。
  2. 前端筛选下拉框缺少 "所有记录" 和 "正常" 选项，导致用户无法切换回查看所有数据。
  3. 如果当前月份只有正常记录或无异常记录，默认列表即为空，用户误以为数据丢失。

## 修复方案
- **修复思路**：
  1. 保持默认只显示异常记录的逻辑（符合业务需求）。
  2. 在前端下拉框增加 "所有记录" (All) 和 "正常" (Normal) 选项，允许用户手动查看这些数据。
  3. 优化前端参数映射，当选择 "所有记录" 时传 `undefined` 给后端，后端则不进行状态过滤。

- **改动文件**：
  - `packages/web/src/pages/attendance/correction/CorrectionProcessingPage.tsx`

- [x] 后端数据验证：通过脚本验证数据库中存在异常记录 (early_leave)，确认数据本身未丢失。
