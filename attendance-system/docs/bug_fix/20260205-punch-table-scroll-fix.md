# 20260205-punch-table-scroll-fix.md

## 问题描述
- **现象**：原始考勤记录列表在列数较多或屏幕较窄时，由于没有横向滚动条，导致部分列被截断无法查看。
- **影响范围**：原始考勤记录页面 (ClockRecordPage)。

## 根因分析
- **直接原因**：
  1. `PunchTable` 组件容器使用了 `min-w-max` 类，导致容器宽度被强制撑大到内容宽度。
  2. 外层父容器使用了 `overflow-hidden`，导致溢出的表格内容被直接裁剪而非显示滚动条。
  3. 表格列未强制禁止换行 (`whitespace-nowrap`)，导致在宽度不足时优先换行增加高度，而非触发横向滚动。

## 修复方案
- **样式调整**：
  1. 移除 `PunchTable` 容器的 `min-w-max`，改为 `w-full` 以适应父容器宽度。
  2. 确保 `PunchTable` 容器保留 `overflow-auto` 以启用滚动条。
  3. 为表格的所有 `th` 和 `td` 添加 `whitespace-nowrap` 类，强制内容不换行，从而在宽度不足时撑开表格触发横向滚动。

## 验证结果
- [x] 表格内容不再换行。
- [x] 当表格宽度超过视口宽度时，出现横向滚动条。
- [x] 页面整体布局未受影响。
