# Bug Fix: 人员管理缺少根部门（全部部门）选项

## 问题描述
用户反馈“人员管理缺少根部门”。经排查，是在人员管理页面的左侧部门树中，仅显示了具体的部门列表，缺少一个可以查看“所有部门”或取消选中的入口。这导致用户一旦选中某个部门后，无法方便地切换回查看所有人员的状态。

## 修复方案
在 `DepartmentSidebar.tsx` 组件中，在部门树的顶部添加一个“全部部门”的静态选项。
- 点击该选项时，触发 `onSelect('')`，传递空字符串作为部门 ID。
- `EmployeeList` 组件接收到空字符串后，会将查询参数 `deptId` 置为 `undefined`，从而获取所有部门的人员。

## 变更文件
- `packages/web/src/pages/employee/components_new/DepartmentSidebar.tsx`

## 验证
- 代码逻辑验证：新增选项符合 `Department` 接口定义，点击事件正确处理了空 ID。
- 类型检查：通过。

## 影响范围
- 仅影响人员管理页面的左侧侧边栏显示，不影响后端 API 或数据库结构。
