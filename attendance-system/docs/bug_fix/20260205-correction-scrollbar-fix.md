# 补签记录滚动条位置修复记录

## 问题描述
- **现象**：在补签记录页面，当无数据时，横向滚动条显示在"暂无记录"图标和文字的上方。
- **复现步骤**：
  1. 进入"补签记录"页面。
  2. 确保当前筛选条件下无数据。
  3. 观察表格区域，横向滚动条（由于表格最小宽度限制）出现在"暂无记录"提示的上方。
- **影响范围**：补签记录页面（CorrectionView）。

## 设计锚定
- **所属规格**：UI-CLONE-Correction
- **原设计意图**：表格应能够横向滚动以展示所有列，无数据时应展示空状态提示。
- **当前偏离**：滚动条层级/位置不符合常规 UX，遮挡或位于空状态提示上方，造成视觉困扰。

## 根因分析
- **直接原因**：空状态组件（No Data div）被放置在滚动容器（`overflow-x-auto`）的外部（下方），但滚动容器因表格设置了 `min-w-[1000px]` 而在视口较窄时出现滚动条。
- **根本原因**：DOM 结构布局问题，空状态应作为表格内容的一部分，或者位于滚动视图内部，以确保滚动条位于所有内容之下。

## 修复方案
- **修复思路**：将空状态提示移入表格的 `tbody` 中，作为一个跨所有列（`colSpan={9}`）的行展示。
- **改动文件**：
  - `packages/web/src/pages/attendance/correction/components/CorrectionView.tsx`

## 验证结果
- [x] 原问题已解决：滚动条现在位于表格（包含空状态提示）的最下方。
- [x] 回归测试通过：有数据时正常展示，无数据时展示空状态且布局正常。
- [x] 设计一致性确认：符合通用 UI 规范。
