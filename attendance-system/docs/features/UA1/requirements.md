# 需求文档：UA1 用户与权限

## 概述
实现基础的用户管理与认证鉴权功能，为考勤系统提供用户身份与组织架构支持。

## User Stories

### Story 1: 用户登录
**As a** 员工, **I want** 登录系统, **So that** 我可以查看考勤和打卡。

**验收标准（AC）：**
- [ ] AC1: 支持用户名/密码登录
- [ ] AC2: 登录成功返回 JWT Token
- [ ] AC3: 登录失败提示明确错误信息

### Story 2: 用户管理
**As a** 管理员, **I want** 管理员工信息, **So that** 维护组织架构。

**验收标准（AC）：**
- [ ] AC1: 创建/编辑/禁用用户
- [ ] AC2: 关联用户部门
- [ ] AC3: 列表查询支持分页和筛选

### Story 3: 个人信息
**As a** 用户, **I want** 查看个人信息, **So that** 确认信息无误。

**验收标准（AC）：**
- [ ] AC1: 获取当前登录用户信息
- [ ] AC2: 修改密码

## 数据模型需求
- 用户 (User): 账号、密码、姓名、部门ID、角色
- 部门 (Department): 名称、父级ID

## 约束条件
- 密码必须加密存储 (bcrypt)
- 接口需鉴权 (JWT)
