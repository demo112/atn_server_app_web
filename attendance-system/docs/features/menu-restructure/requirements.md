# Requirement: 考勤系统菜单重构

## 1. 背景与目标
为了优化用户体验，理顺考勤系统的功能分类，需要对 Web 端的侧边栏菜单进行重构。主要涉及考勤管理和统计报表的分类调整。

## 2. 变更范围

### 2.1 新增菜单组
- **考勤配置** (一级菜单)
  - 包含原“考勤管理”下的设置类功能。

### 2.2 菜单重命名与调整
- **考勤管理** (一级菜单) -> 改名为 **考勤处理**
- **统计报表** (一级菜单) -> 改名为 **考勤统计**
- **统计仪表盘** (二级菜单) -> 改名为 **统计报表**

### 2.3 新增二级菜单
- **打卡记录** (在“考勤统计”下)

## 3. 详细需求

### 3.1 菜单结构定义

| 一级菜单 | 二级菜单 | 对应路径 (参考) | 说明 |
|---------|---------|----------------|------|
| **用户管理** | - | `/users` | 保持不变 |
| **人员管理** | - | `/employees` | 保持不变 |
| **考勤配置** | 考勤设置 | `/attendance/settings` | 从原“考勤管理”移动 |
| | 班次管理 | `/attendance/shifts` | 从原“考勤管理”移动 |
| | 排班管理 | `/attendance/schedule` | 从原“考勤管理”移动 |
| **考勤处理** | 补签处理 | `/attendance/correction-processing` | 从原“考勤管理”移动 |
| | 补签记录 | `/attendance/correction` | 从原“考勤管理”移动 |
| | 请假管理 | `/attendance/leave` | 从原“考勤管理”移动 |
| **考勤统计** | 统计报表 | `/statistics/dashboard` | 原“统计仪表盘”，改名 |
| | 打卡记录 | `/attendance/daily-records` | 新增项 (复用原“每日考勤”功能) |
| | *其他* | *隐藏/移除* | 原“个人汇总”、“统计报表”等未提及项暂不显示 |

### 3.2 路由与页面
- 路由路径保持不变，仅调整菜单的层级展示和名称。
- “打卡记录”指向原“每日考勤”页面 (`/attendance/daily-records`)。
- “统计报表”指向原“统计仪表盘”页面 (`/statistics/dashboard`)。

## 4. 验收标准
1. 侧边栏菜单结构符合上述定义。
2. 点击各菜单项能正确跳转到对应页面。
3. 页面功能正常加载。
