# Requirements: 手机号登录

## Overview

允许用户使用关联员工档案中的手机号（Employee Phone）替代用户名进行登录。支持 Web 端和 App 端。此功能旨在提升用户登录便捷性，无需记忆复杂的用户名。

## User Stories

### Story 1: Web 端手机号登录

**As a** 员工, **I want** 在 Web 端登录页输入手机号和密码登录, **So that** 我可以用手机号快速访问系统

**Acceptance Criteria:**

- [ ] AC1: 支持手机号输入
  - **Given**: 用户已绑定员工档案且档案中有手机号
  - **When**: 在登录页"用户名"输入框输入手机号，密码框输入正确密码，点击登录
  - **Then**: 系统识别手机号，验证通过并成功登录，跳转至首页

- [ ] AC2: 手机号不存在或密码错误
  - **Given**: 输入的手机号未关联任何用户，或密码错误
  - **When**: 点击登录
  - **Then**: 提示"用户名或密码错误"（保持模糊以防枚举）

- [ ] AC3: 多账号共用手机号处理
  - **Given**: 系统中存在多个员工使用相同手机号（异常数据）
  - **When**: 使用该手机号登录
  - **Then**: 登录失败，提示"该手机号关联多个账号，请使用用户名登录"或通用错误提示，后台记录警告日志

### Story 2: App 端手机号登录

**As a** 员工, **I want** 在 App 端使用手机号登录, **So that** 方便移动办公

**Acceptance Criteria:**

- [ ] AC1: App 登录支持
  - **Given**: App 处于未登录状态
  - **When**: 在账号输入框输入手机号和密码
  - **Then**: 成功登录进入 App 首页

## Constraints

- **唯一性约束**：虽然数据库层面 `Employee.phone` 未设置唯一索引，但业务逻辑上手机号登录仅支持**唯一匹配**的手机号。
- **数据关联**：用户必须关联 Employee 档案 (`User.employeeId` 不为空) 且 Employee 档案中有 `phone` 字段。
- **密码一致性**：手机号登录使用与用户名登录相同的密码（`User.passwordHash`）。

## Out of Scope

- **手机验证码登录**：本次仅支持"手机号+密码"，不支持短信验证码。
- **手机号注册**：不提供通过手机号注册新账号的功能。
- **找回密码**：不包含通过手机号找回密码的功能。

## Assumptions

以下假设已与用户确认：
- [x] 登录输入框无需区分"用户名"和"手机号"，由后端自动识别。
- [x] 若手机号与某用户的 `username` 相同（极端情况），优先匹配 `username`。
