# Requirements: 请假管理 - 部门树筛选增强

## Overview

在“请假/出差管理”页面增加部门树筛选功能，支持按部门结构（包含子部门）和人员筛选请假记录，提升管理员查询效率。

## User Stories

### Story 1: 按部门筛选请假记录

As a 考勤管理员, I want 通过部门树选择特定部门, so that 我可以查看该部门（及其子部门）下的所有请假记录

**Acceptance Criteria:**
- [ ] AC1: 页面左侧展示公司部门树结构
- [ ] AC2: 点击部门树节点，右侧列表自动刷新，仅显示该部门及其子部门员工的请假记录
- [ ] AC3: 支持选中"全公司"节点，查看所有记录
- [ ] AC4: 部门树应默认展开一级部门

### Story 2: 部门与人员联动筛选

As a 考勤管理员, I want 在选择部门后进一步选择人员, so that 我可以精确查找特定员工的记录

**Acceptance Criteria:**
- [ ] AC1: 人员选择器应根据当前选中的部门进行过滤
  - 若选中"研发部"，人员列表仅显示研发部员工
  - 若选中"全公司"，人员列表显示所有员工
- [ ] AC2: 人员选择器支持搜索（按姓名或工号）
- [ ] AC3: 选择人员后，列表仅显示该员工的请假记录
- [ ] AC4: 切换部门时，应自动清空已选的非该部门人员

## Constraints

- **UI布局**: 采用左树右表布局，保持界面整洁
- **性能**: 部门树和人员列表应异步加载，避免卡顿
